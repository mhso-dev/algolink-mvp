## 3. 발견 + 수정한 버그 / 약점

### 버그 #1 — 이력서 PDF 생성 500 에러 (제품 버그)

- **현상**: `/me/resume/export?mask=true` GET → HTTP 500, `{"error":"PDF 생성 중 오류가 발생했습니다."}`.
- **로그**: `Error: Could not resolve font for NotoSansKR, fontWeight 400, fontStyle italic` (`@react-pdf/renderer`).
- **원인**: `src/components/instructor/resume-pdf-document.tsx` 의 `empty` 스타일이 `fontStyle: "italic"` 을 사용했지만, `Font.register` 는 Regular / Bold 두 가중치만 등록 — italic variant 미설치.
- **수정**: `src/components/instructor/resume-pdf-document.tsx:55-57` 에서 `fontStyle: "italic"` 제거. 색상(#888888)으로만 강조.
- **검증**: `curl /me/resume/export?mask=true` → HTTP 200, `application/pdf`, 70 KB.
- **영향 범위**: 모든 강사 이력서 PDF 다운로드 (production blocker).

### 버그 #2 — 프로젝트 등록 폼이 빈 날짜 필드를 거부 (제품 버그)

- **현상**: 신규 프로젝트 등록 시 `startAt` / `endAt` 미입력 상태에서 `createProjectAction` 이 `{ ok: false, fieldErrors: { startAt: ["유효한 날짜를 입력해야 합니다."], endAt: [...] } }` 반환.
- **원인**: `src/lib/validation/project.ts` 의 `isoDateLike` 는 `z.preprocess(fn, z.date(...)).optional()` 형태. `optional()` 은 `undefined` 만 통과시키지만, `formData.get("startAt")` 은 빈 문자열 `""` 을 반환 → preprocess 가 `undefined` 로 변환 → 안쪽 `z.date()` 가 `undefined` 거부.
- **수정**: `src/app/(app)/(operator)/projects/new/actions.ts` 에 `blankToUndef()` 헬퍼 추가, 빈 문자열을 `undefined` 로 정규화 후 safeParse 호출.
- **검증**: 신규 프로젝트 등록 happy path 테스트 통과 (`/projects/{id}` 로 redirect).
- **영향 범위**: 모든 프로젝트 신규 등록 (operator, admin) — 시작/종료일을 비워두는 정상 케이스가 막혔음.

### 약점 #3 — 정산요청 알림 트리거가 operator 가 아닌 instructor 에게 dispatch (테스트 설계 결함)

- **현상**: 기존 `phase2-notify` 테스트가 operator 본인의 bell 카운트를 측정 → 항상 0 → skip.
- **원인**: SPEC-NOTIFY-001 §F-206 + `src/lib/payouts/mail-stub.ts` — `settlement_requested` 알림 수신자는 `instructors.user_id` (강사). operator 는 발신자.
- **수정**: 테스트를 cross-role 검증으로 재설계 — operator 가 정산 요청 → 별도 instructor browser context 에서 bell `+1` → dropdown 클릭 후 `-1` 확인. Next.js 라우터 캐시 우회를 위해 cache-bust 쿼리(`?_cb=...`) + 5회 재시도 폴링.

### 약점 #4 — Topbar 로그아웃 셀렉터 부재 (테스트 결함)

- **현상**: `operator can sign out` 테스트가 dropdown 트리거 셀렉터를 잘못 추정 → skip.
- **원인**: `src/components/app/topbar.tsx` 의 `DropdownMenuTrigger` aria-label = `"프로필 메뉴 열기"` (이메일 아님).
- **수정**: 테스트를 `getByRole("button", { name: "프로필 메뉴 열기" })` → 클릭 → `getByRole("menuitem", { name: /로그아웃/ })` → 클릭 으로 단순화.

### 약점 #5 — phase2-admin 비밀번호 환경변수 의존 (테스트 결함)

- **현상**: `SEED_OPERATOR2_PASSWORD` env 미설정 시 무조건 skip.
- **원인**: `helpers/seed-users.ts` 의 `SEED_USERS.operator2.password` 는 시드 SQL 와 동일한 기본값(`DevOperator2!2026`) 을 노출하는데, 테스트는 env 만 참조.
- **수정**: `process.env.SEED_OPERATOR2_PASSWORD?.trim() || SEED_USERS.operator2.password` 로 fallback 추가.

### 약점 #6 — AI 매칭 / 1-클릭 배정 테스트가 시드 ordering 에 종속 (테스트 결함)

- **현상**: `/projects` 첫 행이 이미 강사 배정된 프로젝트(`40000000-...001`) 로 결정 → 추천 버튼 미노출 → skip.
- **수정**: 테스트가 명시적으로 `?status=assignment_review` 필터를 적용해 미배정 프로젝트만 대상으로 함. 1-클릭 배정 회귀와 신규 프로젝트 등록 happy path 를 분리.

### 약점 #7 — 테스트 간 settlements 상태가 누적 mutate 되어 후속 PASS 보장 불가

- **현상**: phase2-payout 이 pending → paid 로 영구 mutate. 재실행 시 pending 시드 고갈 → 알림/payout 테스트 skip.
- **수정**: `tests/e2e/global-setup.ts` 추가. `playwright.config.ts.globalSetup` 등록.
  - 모든 settlements `status='pending'` 으로 reset (`settlement_status_history` 삭제 후 UPDATE)
  - 모든 notifications `read_at=NOW()` 로 정규화 (bell 카운트 기준선 0)
  - `postgres` 모듈 사용 (이미 dependencies 에 존재).

## 6. 잔여 리스크 / 개선 제안

1. **알림 paid 동결 정책의 e2e 우회**
   - SPEC-PAYOUT-001 §M5 paid 동결은 production 보호용. globalSetup 이 e2e 환경에서 이를 우회 (status reset).
   - 의도적 trade-off — production DB 에서는 절대 실행되지 않도록 `process.env.NODE_ENV !== "production"` 가드 추가 권장 (현재는 `DATABASE_URL` 만 체크하나, 향후 staging URL 사용 시 보강 필요).

2. **자동화된 fix loop 전략 검토**
   - 본 작업에서 worktree subagent 격리 대신 인-세션 직접 수정으로 진행 — 컨텍스트 절약 + 검증 즉시성 우선. 향후 수정 패치가 5+ 파일을 건드릴 경우 expert-debug subagent + worktree 격리로 전환 권장.

3. **추가 검증 가능 영역**
   - Concurrency: 동일 settlement 에 두 operator 가 동시에 정산 요청 → optimistic locking 검증 (현재 미커버)
   - 동시 강사 배정 race: 2 operator 가 같은 프로젝트에 1-클릭 배정 → DB 단 unique constraint 검증 (스케줄 EXCLUSION 은 db-verify 에서 검증되나 UI level 미커버)
   - Storage upload: 사업자등록증 / 강사 사진 업로드 e2e (fixture 파일 부재로 본 회귀 범위 외)

4. **TypeScript / lint**
   - `pnpm typecheck` 미실행. 수정한 파일은 모두 단순 변경이라 큰 리스크 없으나, CI 파이프라인 통과 확인 필요.
