# Product: 알고링크 AI Agentic 업무 지원 시스템

## 1. 프로젝트 개요

**제품명**: Algolink AI Agentic Work Platform (코드명: `algolink`)
**의뢰사**: 알고링크 (교육 컨설팅 회사)
**목적**: 교육 프로젝트 의뢰 → 강사 섭외 → 배정 → 진행 → 정산 → 증명서 발급의 전 과정을 AI Agentic 시스템으로 자동화하여, 행정/사무 업무 부담을 기존 대비 **10% 수준**으로 감축하고 담당자가 고객 대응에 집중할 수 있는 환경을 제공한다.

### 1.1 비즈니스 컨텍스트

| 구분 | AS-IS | TO-BE |
|---|---|---|
| 업무 도구 | 이메일, 전화/문자, 구글시트 | AI Agentic 통합 플랫폼 |
| 강사 섭외/배정 | 100% 수작업 | AI 추천 + 1-클릭 배정 |
| 의뢰/섭외/고객대응 소요 시간 | 30~60분 | 5~15분 |
| 진행 상황 파악 | 개별 문의 | 시스템 대시보드 + 메모/쪽지/댓글 |
| 정산/증명서 | 수기 | 자동 |

### 1.2 규모

- 고객사 약 50곳
- 강사진 약 100명
- 동시 진행 교육 프로젝트 수십 건

## 2. 사용자 페르소나

### 2.1 강사 (Instructor)

알고링크에 강의를 제공하는 외부 전문가.

**핵심 니즈**
- 본인 일정 관리 (알고링크 강의 + 개인 일정)
- 이력서 관리 및 증명서 발급
- 정산/회계 처리 (연말정산 지원)
- 강의 제안 수락/거절 빠른 응답

### 2.2 담당자 (PM/Operator, 알고링크 직원)

교육 프로젝트를 운영하는 알고링크 내부 PM.

**핵심 니즈**
- 의뢰 접수 → 강사 추천 → 배정 → 진행 → 정산까지 한 화면에서 관리
- 강사/고객사 데이터베이스 검색·필터·메모
- 일정 충돌, 강사 미응답, 미처리 태스크에 대한 알림
- 자동 메일링 (강사 제안, 정산 요청)

### 2.3 관리자 (Admin)

시스템과 회원·권한을 통제.

**핵심 니즈**
- 회원/권한 관리
- 모든 담당자 업무 가시성
- 매출/매입 통계

### 3.2 Out-of-Scope (MVP 이후)

- 카톡/이메일 → 시스템 자동 의뢰 입력 (외부 채널 연동, MVP는 수동 입력 + AI 파싱만)
- 알림톡(카카오 비즈) 발송
- 전자 계약서 서명
- 알고레터(뉴스레터 발행) 모듈
- B2C 러닝 시스템 / eBook 쿠폰
- 알고링크 홈페이지 "Now Learning" 위젯 연계
- 강사 간 소통 게시판
- API 기반 기관 정보 자동 검색

### 3.3 가정 (Assumptions)

> ⚠️ 다음 결정은 합리적 디폴트로 진행. 사용자 피드백 시 수정 가능.

1. **단일 워크스페이스**: 알고링크 1개 조직 전제(멀티테넌트 X)
2. **언어**: 한국어 단일
3. **타임존**: Asia/Seoul 고정
4. **이메일 발송**: MVP는 콘솔 로그 + 인앱 알림으로 대체(추후 Resend/AWS SES)
5. **결제/세금계산서 발행**: MVP는 데이터 기록만, 실제 국세청 연동은 별도
6. **파일 저장**: Supabase Storage
7. **AI 모델**: Anthropic Claude (Sonnet 4.6 기본, 이력서 파싱은 cache 활용)

## 4. 핵심 사용자 시나리오 (Golden Path)

### 시나리오 A: 강사 온보딩 → 첫 강의 컨펌
1. 신규 강사가 초대 링크로 가입 → 비밀번호 설정
2. 본인 PDF 이력서 업로드 → AI가 양식에 자동 채움 → 검토/수정 후 저장
3. 기술스택/도메인 체크 → 캘린더에 휴가 일정 입력
4. 담당자가 보낸 강의 제안 알림 수신 → 수락
5. 일정에 자동 등록 → D-7, D-1 알림 수신

### 시나리오 B: 담당자 신규 의뢰 → 강사 배정
1. 담당자가 메일로 받은 의뢰 내용을 폼에 입력(또는 PDF 첨부 → AI 파싱)
2. 시스템이 강사 Top-3 추천 (기술스택/일정/만족도)
3. 담당자가 1순위 선택 → 배정 요청 1-클릭 → 강사에게 알림
4. 강사 컨펌 시 상태 자동 전환 → 캘린더 등록
5. 교육 종료 후 만족도 입력 → 정산 요청 1-클릭 → 강사에게 정산 메일

### 시나리오 C: 정산 마감
1. 담당자가 월말 정산 화면에서 미정산 건 일괄 조회
2. 강사별 합계 자동 계산(인건비/세금계산서 분기)
3. 강사가 본인 정산 화면에서 지급 정보 확인 → 입금 후 상태 완료

## 5. 성공 지표 (MVP KPI)

| 지표 | 목표 | 측정 방법 |
|---|---|---|
| 의뢰→배정 평균 소요 시간 | 30분 → 5분 | 상태 전환 타임스탬프 |
| 강사 추천 1순위 채택률 | ≥ 60% | 추천 vs 실제 배정 비교 |
| 이력서 AI 파싱 정확도 | 핵심 필드 ≥ 90% | 사용자 수정률 < 10% |
| 월 정산 처리 시간 | 50% 단축 | 담당자 설문 |

## 6. 제약 사항

- **개인정보 보호**: 강사 주민번호/계좌는 암호화 저장(Supabase RLS + pgcrypto)
- **다운로드 시 마스킹 옵션** 필수
- **회계 정확성**: 정산 금액 계산은 단위 테스트 필수
- **AI 응답 fallback**: Claude API 장애 시 수동 입력 경로 항상 유지
